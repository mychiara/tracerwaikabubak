Persiapan Kebutuhan Sistem

Sebelum memulai, pastikan lingkungan server Anda memiliki perangkat lunak berikut terinstal:

Node.js (v18 ke atas)

Digunakan untuk melakukan instalasi paket dependensi dan melakukan kompilasi bundel production React Vite.

Akun Supabase (Gratis)

Sebagai backend database cloud terintegrasi untuk menyimpan data alumni, kuesioner, dan dokumen kerjasama.

2

Pengaturan Database Supabase

Sistem ini dilengkapi dengan integrasi database Supabase. Jika Supabase offline, aplikasi otomatis beralih ke penyimpanan LocalStorage.

Langkah Setup Supabase:
  1. Buka dashboard Supabase.com dan buat proyek baru.
  2. Setelah proyek selesai dibuat, masuk ke menu SQL Editor pada panel sebelah kiri.
  3. Buat query SQL baru, salin seluruh isi file skema supabase_schema.sql yang ada di root direktori proyek ini, lalu klik Run.
  4. Proses di atas akan otomatis membuat 4 tabel utama: alumni, mitra_kerjasama, pengguna_lulusan, dan alumni_feedback.
skema SQL Supabase SQL 
-- Schema: alumni, mitra_kerjasama, pengguna_lulusan, alumni_feedback
-- Tabel dibuat dengan relasi UUID dan keamanan RLS dinonaktifkan
-- Agar publik dapat melakukan registrasi/input feedback tracer study.
3

Variabel Lingkungan (.env)

Buat file baru bernama .env di root direktori proyek Anda untuk mengkoneksikan backend Supabase:

Isi Berkas .env ENV 
VITE_SUPABASE_URL=https://nama-project-kamu.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJzdXBhYmFzZSIsInJlZiI6InZzY...
PENTING! Pastikan Anda menggunakan prefiks VITE_ pada nama variabel agar dapat dibaca oleh runtime kompilasi Vite pada sisi client.
4

Langkah Kompilasi Produksi (Build)

Lakukan instalasi dependensi modul Node dan kompilasi proyek menjadi file static HTML/JS/CSS:

1. Jalankan instalasi dependensi (hanya sekali):
npm install
2. Jalankan proses compile build untuk production:
npm run build

Setelah proses build selesai, direktori baru bernama "dist" akan terbuat di root folder Anda. Folder ini berisi static assets HTML, CSS, dan JS yang siap diletakkan di web server.

5

Deploy ke Netlify Terkoneksi GitHub

Sesuai arsitektur modern, seluruh script ini disimpan di GitHub dan dideploy secara otomatis menggunakan Netlify:

Langkah 1: Push Project ke GitHub
  1. Inisialisasi git pada folder lokal: git init
  2. Buat repositori baru di GitHub (misalnya: tracer-study-sumba).
  3. Tambahkan semua berkas ke git staging: git add .
  4. Lakukan commit awal: git commit -m "feat: inisialisasi portal tracer study"
  5. Hubungkan repositori lokal ke GitHub dan push:
    git remote add origin https://github.com/username-kamu/tracer-study-sumba.git
    git branch -M main
    git push -u origin main
Langkah 2: Koneksi dan Deploy di Netlify
  1. Buka dashboard Netlify.com dan masuk menggunakan akun GitHub Anda.
  2. Pilih tombol "Add new site" lalu pilih opsi "Import an existing project".
  3. Pilih penyedia layanan Git: GitHub, lalu pilih repositori tracer-study-sumba yang telah dibuat.
  4. Pada bagian Build settings, pastikan konfigurasi berikut sudah sesuai:
    • Build command: npm run build
    • Publish directory: dist
  5. Masuk ke bagian Environment Variables di bawahnya, klik "Add variable" dan masukkan variabel berikut:
    VITE_SUPABASE_URL = [URL_SUPABASE_PROYEK_ANDA]
    VITE_SUPABASE_ANON_KEY = [ANON_KEY_SUPABASE_PROYEK_ANDA]
  6. Klik tombol "Deploy tracer-study-sumba". Netlify akan mulai mengkompilasi dan mempublikasikan aplikasi Anda secara otomatis.
Konfigurasi Penanganan SPA (Single Page Application) Redirects

Agar routing React Router bekerja lancar saat halaman di-refresh di Netlify, sistem ini telah dilengkapi berkas public/_redirects berisi: 

/*    /index.html   200

Berkas ini akan disalin ke folder dist saat proses build dan memberitahu Netlify untuk mengalihkan seluruh rute URL kembali ke file utama index.html.

6

Deploy ke cPanel Traditional & Laragon

Jika Anda menggunakan web hosting konvensional (cPanel) atau server lokal (Laragon/XAMPP):

Deploy ke cPanel:
  1. Buka folder dist di komputer Anda setelah menjalankan npm run build.
  2. Kompres seluruh isi folder dist menjadi satu berkas bernama dist.zip.
  3. Buka cPanel hosting Anda, masuk ke **File Manager** dan buka direktori utama domain (public_html).
  4. Unggah berkas dist.zip lalu ekstrak berkas tersebut langsung di dalam folder public_html.
  5. Buat file bernama .htaccess di folder tersebut untuk mencegah error 404 saat halaman di-refresh:
<IfModule mod_rewrite.c>
  RewriteEngine On
  RewriteBase /
  RewriteRule ^index\.html$ - [L]
  RewriteCond %{REQUEST_FILENAME} !-f
  RewriteCond %{REQUEST_FILENAME} !-d
  RewriteRule . /index.html [L]
</IfModule>
Deploy ke Laragon:
  1. Buka direktori instalasi Laragon Anda (biasanya di C:\laragon\www\nama-folder).
  2. Salin seluruh berkas yang berada di dalam folder dist ke folder proyek Laragon Anda tersebut.
  3. Akses domain lokal yang disediakan oleh Laragon (misalnya http://nama-folder.test).
7

Penyelesaian Masalah (Troubleshooting)

Masalah 1: Kenapa data kuesioner tidak terupdate ke Supabase?

Pastikan Anda sudah mengaktifkan izin tabel publik (tanpa RLS) atau telah menyalin schema pemicu SQL dengan benar. Jika terjadi kegagalan koneksi, data akan disimpan ke penyimpanan lokal web browser (LocalStorage).

Masalah 2: Blank page / Error 404 ketika navigasi di-refresh di cPanel?

Hal ini wajar pada aplikasi Single Page Application (SPA). Buat berkas .htaccess seperti di Langkah 6 untuk memetakan seluruh rute navigasi kembali ke berkas index.html.

8

Mencegah Hibernasi Database (Supabase Keep-Alive)

Layanan gratis (Free Tier) Supabase akan otomatis masuk ke mode jeda (hibernasi) jika tidak ada aktivitas query dalam waktu 7 hari. Untuk mencegah hal ini, sistem ini telah dilengkapi dengan workflow otomatis **GitHub Actions** yang akan melakukan ping query ringan sekali sehari secara otomatis.

Konfigurasi Otomatis via GitHub Actions

Berkas alur kerja sudah tersedia di folder .github/workflows/supabase-keepalive.yml. Untuk mengaktifkannya, ikuti langkah berikut:

  1. Push proyek Anda ke repositori GitHub.
  2. Buka halaman repositori Anda di GitHub, lalu klik tab Settings di menu atas.
  3. Pada panel sebelah kiri, pilih Secrets and variables -> klik Actions.
  4. Klik tombol "New repository secret" dan tambahkan dua kunci rahasia berikut:
    Kunci 1:
    Name: VITE_SUPABASE_URL
    Value: [Masukkan URL proyek Supabase Anda]

    Kunci 2:
    Name: VITE_SUPABASE_ANON_KEY
    Value: [Masukkan Anon Key proyek Supabase Anda]
  5. Setelah disimpan, GitHub Actions akan menjalankan ping query otomatis setiap hari pada pukul 13:00 WITA. Anda juga bisa mengujinya secara manual dengan pergi ke tab Actions di repositori GitHub Anda, pilih workflow Supabase Keep Alive, lalu klik Run workflow.
